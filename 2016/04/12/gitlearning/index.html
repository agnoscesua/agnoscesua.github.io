Git学习笔记

发表于 2016-04-12

Git是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

Git的安装配置

LINUX 平台安装

Windows 平台安装

首先下载 Git

安装成功后会发现鼠标右击，会多出两行Git GUI Here 和 Git Bash Here

Git 提供了一个叫做 git config 的工具，专门用来配置或读取相应的工作环境变量。

配置SSH

先检查本机是否有ssh key的配置(Git Bash)

$ cd ~/.ssh

若没有则会提示：No such file or directory

若有则进入该路径，ls查看当前路径下所有文件，rm * 删除所有文件

接下来生成新的ssh key

$ cd ~
$ ssh-keygen -t rsa -C "youremail@xxx.com"

此时会出现如下内容：

Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/xxxx_000/.ssh/id_rsa):   //不填直接回车
Enter passphrase (empty for no passphrase):   //输入密码（可以为空）
Enter same passphrase again:   //再次确认密码（可以为空）
Your identification has been saved in /c/Users/xxxx_000/.ssh/id_rsa.   #生成的密钥
Your public key has been saved in /c/Users/xxxx_000/.ssh/id_rsa.pub.  #生成的公钥
The key fingerprint is:

此时新的ssh key已经生成好。

添加ssh key到GitHub

登录到GitHub,然后点击右上角账号头像的“▼”，选择Settings -> SSH and GPG keys -> New SSH key

Title自定义，然后复制公钥，C:\Users\XXXX\.ssh目录下的id_rsa.pub全部内容，最后点击Add key

配置用户名和电子邮件
$ git config --global user.name "yourusername" //github用户名
$ git config --global user.email "youremail@xxx.com" //github注册邮箱
测试ssh keys是否设置成功
$ ssh -T git@github.com

此时会出现如下内容：

The authenticity of host 'github.com (192.30.252.129)' can't be established.
RSA key fingerprint is 16:27:xx:xx:xx:xx:xx:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)? yes  //确认你是否继续联系，输入yes
Warning: Permanently added 'github.com,192.30.252.129' (RSA) to the list of known hosts.
Enter passphrase for key '/c/Users/xxxx_000/.ssh/id_rsa':  //生成ssh kye是密码为空则无此项，若设置有密码则有此项且，输入生成ssh key时设置的密码即可。
Hi xxx! You've successfully authenticated, but GitHub does not provide shell access. //出现这句话，说明设置成功。
将本地项目push到GitHub远程仓库

创建本地仓库，自己新建也可以

$ mkdir testgit

进入本地仓库目录下

$ cd testgit

初始化

$ git init

新建md文件，不用命令建也可以

$ echo "Hello world" >hello.md

提交到本地,对该文件追踪

$ git add hello.md

提交说明

$ git commit -m "my first commit"

提交到GitHub

$ git remote add origin 粘贴你复制的要提交仓库的ssh链接(git@github.com:xxxxxxx/yyyyyy.git)
$ git push -u origin master

若此时出现错误，无法提交，则输入如下命令

$ git fetch origin master   //一旦远程主机的版本库有了更新(Git术语叫做commit)，需要将这些更新取回本地，这时就要用到git fetch命令
$ git pull origin master

最后再输入

$ git push -u origin master

刷新查看远程仓库，提交成功!
